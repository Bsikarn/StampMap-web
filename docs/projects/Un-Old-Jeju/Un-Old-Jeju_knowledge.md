# ฐานความรู้เทคนิคที่ใช้ (Un-Old-Jeju Knowledge)

เอกสารนี้รวบรวม "รายชื่อเทคนิคการเขียนโปรแกรมและการจัดการระบบ" ขั้นสูงที่ถูกนำมาใช้ภายในโปรเจกต์ Un-Old-Jeju โดยเฉพาะ 

### 1. Intercepting Routes (@modal) & Parallel Routes
- **บริบทที่ใช้:** การทำหน้า `<LocationModal />` เมื่อผู้ใช้กดที่เข็มหมุดบนหน้า Landing Page
- **วิธีการทำงาน:** เราใช้โครงสร้างโฟลเดอร์แบบ `app/@modal/(.)location/[id]/page.tsx` ใน Next.js โดยถ้าผู้ใช้กดจากหน้าแรก (/) มันจะเอาหน้า `/location` มาสวมทับ (Overlay) ในตัวแปร `{modal}` บนหน้าแรกทันทีโดยที่ฉากหลัง (แผนที่) ยังคงอยู่ แทนที่จะพุ่งเปลี่ยนไปหน้าขาวๆเปล่าๆ แต่ถ้าผู้ใช้อันนั้น Copy Link ส่งให้เพื่อน เมื่อเปิดเว็บเข้ามาใหม่ มันจะวิ่งลงไปที่ `/location/[id]/page.tsx` ตัวหลักแทน

### 2. Component Extraction & DRY Principle กรองโค้ดซ้ำกระชับหน้า
- **บริบทที่ใช้:** การรวมร่างระหว่าง Mock Data สองหน้าจอให้ใช้จุดเดียวกัน (หน้าหลัก, หน้า Settings, หน้า Souvenir)
- **วิธีการทำงาน:** เราทำการแยก (Extract) HTML ก้อนใหญ่ๆ ออกมาเป็น `<SouvenirCard />`, `<ExchangeModal />`, `<SettingsGroup />` และนำไปเก็บในโฟลเดอร์ `components/` ของตัวโปรเจกต์ การทำเช่นนี้ทำให้ไฟล์ `page.tsx` มีจำนวนบรรทัดที่สั้นลง อ่านเข้าใจง่าย และสามารถนำ `StarRating` ไปประกอบร่างใช้ซ้ำได้ในหน้าอื่นๆทันที

### 3. Gesture Physics & Pointer Events เด้งคืนกลับ (Drag to Dismiss)
- **บริบทที่ใช้:** การดึงเลื่อนหน้าจอหน้ารายละเอียดสถานที่ เมื่อลากนิ้วจากตำแหน่งหัวสุดเพื่อปิด Modal
- **วิธีการทำงาน:** เราใช้งาน `onPointerDown`, `onPointerMove`, และ `onPointerUp` ทำการจับตำแหน่ง `clientY` ของนิ้ว หากผู้ใช้ลากแผง Modal ดึงลงต่ำกว่าระยะ Threshold หน้าต่างจะถูกสั่ง `router.back()` เพื่อปิดออก แต่ถ้าลากลงไม่สุดระบบจะสั่ง Re-render State คืนหน้าต่างไปที่เลข 0 ทำให้หน้าจอมีความรู้สึกเหมือนยางยืดที่เด้งกลับมาแบบ Mobile App ของแท้

### 4. Zustand Immutable State Management 
- **บริบทที่ใช้:** การจัดเก็บจำนวนและชิ้นส่วนของ Souvenir/Stamps ที่สะสมมาได้ (`useAppStore`, `useStampStore`)
- **วิธีการทำงาน:** เวลาเรียก State ปกติของ React มันจะทำลายและสร้างตัวเองใหม่เมื่อข้อมูลถูกแก้ ในการจัดการ Data ขนาดใหญ่ข้ามหน้าเราเลือกใช้ Zustand แทน Context API เพราะเบากว่า โดยตั้งกฏหลีกเลี่ยงการ Mutate ตัวแปรตรงๆ หากจะอัพเดตค่า List จะใช้วิธี `collectedStamps: [...state.collectedStamps, newStamp]` ในการกระจาย Array เดิมและโอบ Array ใหม่ไว้
